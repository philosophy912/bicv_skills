# Jenkins REST API 安全确认矩阵

执行 Jenkins 写操作前，必须先按下表判断风险等级。所有 `严重` 和 `高危` 操作都必须逐次征得用户明确确认。

## 风险分级

| 风险等级 | 判定标准 | 执行要求 |
|---------|---------|---------|
| 低 | 纯只读查询，不改变 Jenkins 状态 | 可直接执行 |
| 中 | 会改变运行态但可快速恢复，且影响面明确 | 默认先提醒影响；若用户请求已足够明确，可执行 |
| 严重 | 会中断任务、改变可用性，或触发真实流水线 | 必须先确认 |
| 高危 | 会删除、覆盖配置、创建新资源，或产生不可逆后果 | 必须先确认，且要明确影响范围 |

## 常见操作矩阵

| 操作 | 方法 | 端点模式 | 风险等级 | 原因 |
|------|------|---------|---------|------|
| 列出 Jobs / 查询构建 / 查看日志 / 查看队列 | `GET` | `/api/json`、`/job/*/api/json`、`/consoleText`、`/queue/api/json` | 低 | 纯读取 |
| 获取 Crumb | `GET` | `/crumbIssuer/api/json` | 低 | 纯读取 |
| 触发无参构建 | `POST` | `/job/{job}/build` | 严重 | 会启动真实流水线 |
| 触发带参构建 | `POST` | `/job/{job}/buildWithParameters` | 严重 | 会启动真实流水线，且参数可能扩大影响 |
| 停止构建 | `POST` | `/job/{job}/{build}/stop` | 严重 | 会中断正在执行的构建 |
| 禁用 Job | `POST` | `/job/{job}/disable` | 严重 | 会影响后续构建可用性 |
| 启用 Job | `POST` | `/job/{job}/enable` | 中 | 会恢复构建能力，仍会改变系统状态 |
| 删除 Job | `POST` | `/job/{job}/doDelete` | 高危 | 删除资源，可能难以恢复 |
| 创建 Job | `POST` | `/createItem` | 高危 | 新增资源并改变系统结构 |
| 更新 Job 配置 | `POST` | `/job/{job}/config.xml` | 高危 | 可能覆盖配置并引入生产风险 |

## 确认话术要求

确认前至少说明以下信息：

1. HTTP 方法与端点模式
2. 目标 Jenkins 系统和目标 Job / Build
3. 预期影响，例如“会触发真实流水线”或“会停止正在运行的构建”
4. 是否不可逆，或恢复成本是否较高
